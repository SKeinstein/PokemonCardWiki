| 提案サブタグ | 名前件数 | 代表カード | 判定パターン |

| **トレーナーズ** | 5 | デラックスボム / パンクメット（どうぐ） / ロケット団のびっくりボム（グッズ） / スパイクエネルギー（特殊E） / 危ない廃墟（スタジアム） | カード種別がトレーナーズ系 |

**※** 移動型8名は `ベンチ狙撃>移動型`（20 cardId / 8名）と完全一致。スクリプト集計上カウントが7になる場合はデスカーン（2エフェクト）の最初に検出されるテキストの差による処理順依存（実質8名）。

### サブタグ設計メモ

- 移動型は `ベンチ狙撃>移動型` と重複しているため、追加サブタグを付けるなら `ダメカン操作>移動型` とし、両方保持する（既存方針に従う）。
- 直接のせ型の中に「自傷→強化型」（オーダイル: 自分に5個のせ→+120ダメージ）が混在。件数は1件のみのためサブタグ不要か要検討。
- トレーナーズはカード種別で自明なため、サブタグなし（または `ダメカン操作>トレーナーズ`）どちらでも可。

## 4. 条件ダメージ パターン漏れ調査

### 現行サブタグと検出パターン概要

| サブタグ | cardId件数 | 主な検出パターン |
|---|---|---|
| 条件ダメージ>場参照 | 337 | ポケモン数×, エネルギー数×, ダメカンのってるなら, ベンチに〇がいるなら, スタジアムがあるなら, ロケット団/未来サポート条件 |

### 発見された漏れパターン（計 ~23 ユニーク名）

#### Gap-1: トラッシュコスト×ダメージ（16件）

「エネルギーをトラッシュし、その枚数×●ダメージ」型。現在は `エネ除去>自己コスト` のみでタグ付けされ、`条件ダメージ` が付与されていない。

**対象カード**: ウミトリオex / タケルライコex / ハッサムex / ヌオー / マグカルゴex / ヤバソチャ / ヒードラン / サンダースex / ヒビキのマグカルゴ / カスミのギャラドス / ロケット団のミュウツーex / メガユキノオーex / メガディアンシーex / メガリザードンXex / メガピクシーex / クレベース

**追加提案パターン（tag_cards.mjs）**:
```
/トラッシュし.{1,25}(その|の数|枚数)[×x]\d+ダメージ/.test(all)
```
→ `条件ダメージ`, `条件ダメージ>場参照`（トラッシュしたエネルギー枚数参照）

#### Gap-2: トラッシュ参照×ダメージ（4件）

「自分/相手のトラッシュにある〇〇の枚数×ダメージ」型。

**対象カード**: トドロクツキ（古代カードの枚数×10）/ ソウブレイズex（トラッシュのエネルギーの枚数×20）/ Nのヒヒダルマ（相手トラッシュの基本エネルギーの枚数×30）/ ヒビキのバクフーン（「ヒビキの冒険」の枚数×60）

**追加提案パターン**:
```
/トラッシュにある.{0,20}(の数|枚数)[×x]\d+ダメージ/.test(all)
```
→ 自分トラッシュは `条件ダメージ>場参照`、相手トラッシュは `条件ダメージ>相手参照`

#### Gap-3: 手札枚数×ダメージ（2件）

「相手の手札の枚数×●ダメージ」型。現在 `手札干渉>手札参照` のみ付与。

**対象カード**: シャンデラ（相手の手札の枚数×30）/ メガユキメノコex（相手の手札の枚数×50）

**追加提案パターン**:
```
/手札の枚数[×x]\d+ダメージ/.test(all)
```
→ `条件ダメージ`, `条件ダメージ>相手参照`

#### Gap-4: 山札公開枚数×ダメージ（1件）

「山札上●枚を公開し、〇〇のカードの枚数×●ダメージ」型。現在 `未来参照` のみ付与。

**対象カード**: テツノイバラ（山札上5枚公開し「未来」のカードの枚数×70）

**追加提案パターン**:
```
/山札.{0,15}(公開|オモテ).{1,20}(の数|枚数)[×x]/.test(all)
```
→ `条件ダメージ`, `条件ダメージ>場参照`

### 漏れ件数サマリー

| パターン種別 | ユニーク名前数 | cardId概算 |
|---|---|---|
| Gap-1: トラッシュコスト×ダメージ | 16 | ~40 |
| Gap-2: トラッシュ参照×ダメージ | 4 | ~10 |
| Gap-3: 手札枚数×ダメージ | 2 | ~6 |
| Gap-4: 山札公開枚数×ダメージ | 1 | ~4 |
| **合計（重複なし）** | **23** | **~60** |

### 影響評価

修正後の `条件ダメージ` 件数予測:
- 現在: 567 cardId / 282名
- 修正後: ≈ 627 cardId / ~305名（+~60 cardId / +~23名）

サブタグ内訳への影響:
- `条件ダメージ>場参照` が最大の増加（Gap-1: 16名 + Gap-2一部 + Gap-4: 1名）
- `条件ダメージ>相手参照` は Gap-2一部 + Gap-3: 2名で小幅増加
