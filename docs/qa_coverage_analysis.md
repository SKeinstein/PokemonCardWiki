## 3. タグ体系の粒度ミスマッチ

### ミスマッチ例1: QA側に親タグのみ

QAエントリー側: `ポケモン入れ替え`（15件）→ カード側に `ポケモン入れ替え>ワザ`（29枚）があっても親タグは一致しない（マッチングはサブタグのみ）。

### ミスマッチ例2: QA側に存在しないカード専用サブタグ

カード分類として有効だが、汎用的なルールQ&Aでは質問にならないため、QA側にタグが付かない。例:
- `条件ダメージ>コイン可変`（コイントスで変動するダメージ）→ 基本ルールQAに相当するが個別カードQAは少ない
- `ワザロック>自己ロック`（「次の自分のターン、このワザは使えない」系）→ 独立した汎用QAが存在しない

### ミスマッチ例3: 新キャラクター連動の未反映

`キャラクター連動>ナンジャモ`（23枚）等、QAデータ収録時期より後のカードに関するキャラクター連動がQAエントリーに未タグ。QAデータ（XY規制）にナンジャモQAは存在しないか、存在しても未タグ。

## 4. 改善パターンと推定効果

### パターン1（高優先）: QAエントリーへのサブタグ付与

**対象**: B2 の 682 枚
**作業**: `tag_qa_entries.mjs` のタグルールを拡張し、各サブタグをQAエントリーに付与する
**推定上限効果**: +682 枚（最大 52.2% まで改善）

具体的なQAエントリー候補:

| 付与すべきサブタグ | 候補QAエントリー（未タグ） |
|-------------------|------------------------|
| `条件ダメージ>場参照` | #77（ベンチ弱点抵抗力）、#287（カウンターゲイン＋ベンチ狙撃）、#352（はなのカーテン＋ベンチ）、#697（げきりゅうポンプ） |
| `ダメージ操作>抵抗無視` | #27（抵抗力とは）、#73（弱点・抵抗力・すべての効果に関係なく）、#77（ベンチ抵抗力計算）、#81（抵抗力なし）、#108（自分自身へのダメージ） |
| `ポケモン入れ替え>ワザ` | #63（入れ替え後のワザ）、#85（特性後の入れ替えとワザ）、#189（ワザ使用後の入れ替え）、#376（つきとばす＋特性）、#388（トゲトゲローリング再使用） |
| `グッズサポートロック>グッズロック` | 該当するルールQAを検索・確認が必要 |
| `ワザロック>にげられない` | #にげる関連のQAエントリーを確認 |
| `手札干渉>トラッシュ` / `手札を見る` | 既存の `手札干渉>山札戻し`（2件）に隣接するエントリー |

### パターン2（中優先）: B1カードへのサブタグ追加

**対象**: B1 の 488 枚
**作業**: `tag_cards.mjs` のタグルールを拡張し、以下の分類を追加

| 現在 | 追加すべきサブタグ | 対象カード例 |
|------|------------------|-------------|
| `ポケモン入れ替え` | `>グッズ` / `>サポート` / `>ワザ` / `>特性` | グッズ系入れ替えカード |
| `進化加速` | `>山札から` / `>手札から` | スーパーボールなど |
| `自傷` | `>ダメカン` / `>HPコスト` | 自傷ワザ持ちポケモン |
| `エネ移動` | `>自分のみ` / `>相手から` | エネ移動ワザ/特性 |
| `ACE SPEC` | `>制限` | ACE SPECカード全般 |
| `ダメカン操作` | `>移動` / `>除去` / `>追加` | ダメカン移動系 |

**注意**: サブタグ追加後、それらがQAエントリー側にも付与されていないと依然マッチしない（パターン1との連携が必要）。

### パターン3（低優先）: マッチングロジックの拡張

**対象**: B1 の 488 枚（別アプローチ）
**作業**: `build_qa_index.mjs` の relatedQA マッチング条件を拡張し、**トップレベルタグの親一致**も許可する

```js
// 現行: サブタグ完全一致のみ
if (!tag.includes('>')) continue;

// 拡張案: トップレベルタグでも親カテゴリとして一致
// QAエントリーのタグ「ポケモン入れ替え」→ カードの「ポケモン入れ替え>ワザ」にヒット
```

**デメリット**: マッチングが粗くなり、無関係なカードへの「関連QA」が増える可能性がある。

### パターン4（要調査）: 区分A（タグなし）カードへのタグ付け強化

**対象**: 2,008 枚
**現状の課題**: トレーナーズカード（グッズ/サポート）はテキスト情報がJSON内に存在せず、テキストベースの自動タグ付けができない。

**対応案**:
- カード名リストから名前マッチングでタグを付与（例: 「ナンジャモ」→ `キャラクター連動>ナンジャモ`）
- PTCG公式サイトからトレーナーズのテキストをクロールして補完
- 規制セットやカード種別でのバルクタグ付与（例: 全スタジアムに `スタジアム` トップレベルタグ）

### 残課題分析

`ポケモン入れ替え>グッズ`（165枚）の QA エントリー側に 0 件しか付与されなかった理由:
- QA データ（XY 規制）には「ポケモンいれかえ」を直接引用する専用 Q&A が存在しない
- これらのカードは directQA か、別のタグ（`ポケモン入れ替え>ワザ`等）経由でのみ到達可能

`自傷>ダメカン/ダメージ`（80枚）が改善しなかった理由:
- QA エントリーがカードテキストをそのまま引用せず、文体の乖離が大きい
- 例: カードテキスト「このポケモンにも30ダメージ」→ QA 文「自分のバトルポケモンにもダメージを与えるワザ」

`ワザロック>自己ロック`（70枚）が依然 0 件の理由:
- 具体的な Q&A が QA データ（XY 規制）に存在しない可能性が高い
