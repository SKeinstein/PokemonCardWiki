## §2  パターンマッチ問題の原因特定

### 2-1  マギアナ — 誤付与: `エネ加速` / `エネ加速>手札→ポケモン`

**実際のカードテキスト（特性「オートヒール」）:**
```
このポケモンがバトル場にいるかぎり、自分の手札からエネルギーをポケモンにつけるたび、
そのポケモンのHPを「90」回復する。
```

**マッチしたパターン（§5-C branch 2, tag_cards.mjs:224）:**
```javascript
/手札からエネルギーを.{0,10}ポケモンにつける/
```

**根本原因:**  
このパターンは「誰がエネルギーを張る行為の主体か」と「実際にエネルギーを追加で張れるか」を問わない。マギアナの場合、テキストは「手張りを発動条件とするヒール特性」のトリガー節を記述しているだけであり、マギアナ自身がエネルギーを加速する効果は一切ない。パターンが「エネルギー加速効果」と「エネルギー張りを参照する効果」を区別できていない。

**影響:** 3バリアント × 2タグ（エネ加速 / エネ加速>手札→ポケモン）が誤付与  
**正しいタグ:** `ドロー`, `ドロー>そのまま`, `回復`, `回復>固定HP回復`

---

### 2-2  エネルギーリサイクル — 誤付与: `エネ加速` / `エネ加速>トラッシュ→ポケモン`

**実際のカードテキスト（新テキスト、rules フィールド）:**
```
自分のトラッシュから基本エネルギーを5枚まで選び、相手に見せて、山札にもどして切る。
```

**マッチしたパターン（§5-D catch-all, tag_cards.mjs:237）:**
```javascript
/自分のトラッシュから基本エネルギー/
```

**根本原因:**  
§5-D の catch-all パターン `/自分のトラッシュから基本エネルギー/` が「トラッシュ→山札」と「トラッシュ→ポケモン直接装着」を区別しない。エネルギーリサイクルはトラッシュから山札に戻すカードであり、ポケモンに直接つける効果は持たない。除外条件 `!/トラッシュから...手札に加える/` も手札経由のみを除外しており、山札戻し型は未対策。

**影響:** 8バリアント（新テキスト版のみ; 旧テキスト「トラッシュにある基本エネルギー」4件は別パターンのためエネ加速系は付与されない）  
**正しいタグ:** `トラッシュ回収`, `トラッシュ回収>エネルギー`（旧テキスト版も同じ）

---

### 2-3  ビワ — 誤付与: `ドロー` / `ドロー>シャッフル後` / `手札干渉>山札戻し`

**実際のカードテキスト（rules フィールド）:**
```
相手の手札を見て、その中からグッズを2枚まで選び、トラッシュする。
```

**誤付与原因 A（tag_cards.mjs:264）:**
```javascript
name === 'ビワ'  // §B-1-3 ドロー>シャッフル後 ルール
```

**誤付与原因 B（tag_cards.mjs:131）:**
```javascript
['ジャッジマン', 'ビワ'].includes(name)  // §3-3 手札干渉>山札戻し ルール
```

**根本原因:**  
ジャッジマン（「両者の手札を山札に戻してシャッフル・引き直し」）とビワを同一 name-based ルールにまとめた設計ミス。ビワは「相手の手札を見てグッズをトラッシュ」するカードであり、ドロー効果も手札の山札戻しも持たない。テキストが `rules[]` に存在する（規制後テキストとして記録済み）にもかかわらず、name-based fallback が優先して誤ったタグを付与している。

**影響:** 4バリアント × 3タグ（ドロー / ドロー>シャッフル後 / 手札干渉>山札戻し）が誤付与  
**正しいタグ:** `手札干渉`, `手札干渉>手札を見る`, `手札干渉>トラッシュ`

---

### 2-4  レガシーエネルギー — 誤付与: `トラッシュ回収` / `トラッシュ回収>自由選択`

**実際のカードテキスト（rules フィールド）:**
```
このカードをつけているポケモンが、相手のポケモンからワザのダメージを受けてきぜつしたとき、
とられるサイドは1枚少なくなる。対戦中、自分の「レガシーエネルギー」のこの効果は、1回しかはたらかない。
```

**誤付与原因（tag_cards.mjs:372）:**
```javascript
name === 'レガシーエネルギー'  // §B-1-T3 トラッシュ回収>自由選択 ルール
```

**根本原因:**  
`name === 'レガシーエネルギー'` という name-based ルールが §B-1-T3（トラッシュ回収>自由選択）に誤って追加されている。レガシーエネルギーはきぜつ時のサイド枚数を減らすACE SPECエネルギーであり、トラッシュからカードを回収する効果を一切持たない。コメント（tag_cards.mjs:819 "§B-1-T3 にて name-based 付与済み"）も誤った付与を既成事実として記録している。

**影響:** 4バリアント × 2タグ（トラッシュ回収 / トラッシュ回収>自由選択）が誤付与  
**正しいタグ:** `ACE SPEC`, `サイド操作`

---

### 2-5  ピュール — 問題なし

**実際のカードテキスト（rules フィールド）:**
```
自分の手札が5枚になるように、山札を引く。のぞむなら、山札を引く前に、自分の手札を好きなだけトラッシュする。
```

**マッチしたパターン（§B-1-4, tag_cards.mjs:271）:**
```javascript
/手札が\d+枚になるように.{0,15}山札を引く/
```

**判定:** テキストが `rules[]` に正しく記録されており、パターンマッチ正常。付与タグ `ドロー` / `ドロー>固定枚数まで` は意味的に正確。name-based fallback がないが、テキスト経由で完全に捕捉できているため問題なし。
