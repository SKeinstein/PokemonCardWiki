## 2. 各サイトの機能分析

### 2-A. 公式カード検索（pokemon-card.com/card-search）

**強み**
- 最新カード即日収録（公式のため）
- レギュレーション別フィルター（スタンダード / エクストラ / 殿堂 / 全て）
- 進化段階・特殊ルール（ex, V, VMAX, VSTAR 等）フィルター
- HP範囲・にげるコスト・レアリティ・イラストレーター検索
- タイプ（弱点・抵抗力含む）フィルター
- カード種別（グッズ / サポート / スタジアム / ポケモンのどうぐ）フィルター
- カード画像付き表示

**弱み / ギャップ**
- テキスト全文検索なし（ワザ効果テキストで絞り込み不可）
- メカニクスグループ検索なし（「ベンチ狙撃」「入れ替え」等のカテゴリ不在）
- Q&A / ルール情報と検索結果の連携なし
- 旧レギュレーションカードは限定的
- OR / NOT 検索非対応

---

### 2-B. 公式FAQ検索（pokemon-card.com/rules/faq）

**強み**
- 公式回答のため信頼性最高
- カード名 / テキストによるキーワード検索
- レギュレーション・シリーズ別絞り込み
- 「カードの特性やワザの解説も掲載」

**弱み / ギャップ**
- カードDB と独立（FAQ からカード詳細に飛べない）
- カード名 / テキスト検索のみ（メカニクスカテゴリ検索なし）
- 1 Q&A エントリーに複数カードが絡む場合の視覚化なし
- 「このカードに関するQ&A一覧」をカード詳細から参照できない

---

### 2-B-2. pnote（pnote-928a5.web.app / モバイルアプリ）

**概要:** 個人開発の無料ポケモンカード管理アプリ（iOS App Store / Google Play）。広告なし。ランディングページ `pokemon-and-note.com` はDNS障害中だが本体アプリは現役稼働中。

**強み**
- デッキ作成・編集：ドラッグ＆ドロップで直感的にカードを配置
- 大会結果チェック：シティリーグ等の大会結果を参照してデッキ構築
- 対戦記録：デッキ毎に日々の勝敗・詳細記録を保存・分析
- **1人回し機能（ひとり回し）**：ソロシミュレーション対応（ベンチ枠5体上限あり）
- **初手確率計算**：特定カードの初手引き確率をシミュレーション（先攻/後攻切替対応）・グラフ表示
- **スプリット画面で2デッキ同時テスト**：公式ツールにない機能
- **よく使うサポートのショートカット**：ナタネの活気・ボスの指令等を1タップで実行
- **コイントス機能**：シミュレーション内に統合
- **QRコードでデッキ共有**：デッキコードのQR生成・読込対応（画像付き対戦報告のQRも生成）
- **大会形式対応**：BO1 / BO3 / チーム戦の対戦記録に対応
- **対戦成績ダッシュボード**：マッチアップ別勝率・詳細な勝敗分析
- デッキ作成・編集：ドラッグ＆ドロップ＋デッキコードのインポート/エクスポート
- **V-UNIONカード対応**：4枚を自動グループ化
- カード検索：カード名・効果テキスト（ワザ効果含む）で検索可能
- カード分類フィルター：ポケモン / トレーナーズ / エネルギー 大分類
- 収録パック・進化チェーン情報を表示
- 重複カード除外フィルター（最近追加）
- 完全無料・無広告

**弱み / ギャップ（我々との差）**
- **Web非対応**：モバイルアプリ専用のため PC での詳細調査・デッキ設計が困難
- **メカニクスグループ検索なし**：「ベンチ狙撃」「エネ加速」等のカテゴリ検索不可
- **Q&A / ルール統合なし**：カード詳細から公式Q&Aへのリンクなし
- **ルール根拠のサブカテゴリなし**：C-04型 vs C-05型、ダメカン型 vs ダメージ型の区別なし
- **AND / OR / NOT 検索非対応**：複合テキスト検索不可
- レギュレーション別詳細フィルター（HP範囲・攻撃コスト・ダメージ量等）は不明・限定的

**ポジション:** デッキ管理・大会記録・ソロ回し特化。「カードを覚える・理解する」ではなく「デッキを回す・記録する」ツール。

---

### 2-B-3. ポケカ検索くん（pokeca-kensaku-kun.com）

**概要:** 個人開発の高機能カード検索専門サイト。我々のコンセプトに最も近いテキスト検索機能を持つ。

**強み**
- カード名・カードテキスト全文検索（ワザ効果・特性効果を個別に絞り込み可）
- HP範囲・にげるコスト・タイプ（弱点・抵抗力含む）フィルター
- レギュレーション別検索（スタンダード / エクストラ / 殿堂 + スタンダード落ち予測）
- シリーズ / 収録商品 / エキスパンションマーク別フィルター
- 進化段階・特殊ルール（ex, V, VMAX, VSTAR 等）フィルター
- 能力タイプ絞り込み（特性 / ポケパワー / ポケボディー / テラスタル / 古代特性）
- ワザエネルギーの前方 / 完全一致検索
- イラストレーター検索
- 特殊ルールなしポケモンのみ検索
- URLパラメータで検索結果共有可能

**弱み / ギャップ（我々との差）**
- **メカニクスグループ検索なし**：細かいフィルターは豊富だが「ベンチ狙撃まとめ」等のキュレーテッドグループ概念がない
- **Q&A統合なし**：ルール情報との連携ゼロ
- **AND / OR / NOT 複合テキスト検索** の明示的サポートなし（単語検索が主体）
- **ルール解説コンテキストなし**：「なぜこのグループが重要か」の説明なし
- トレーナーズのメカニクスタグ付けなし

**ポジション:** フィルター検索の深さは業界トップレベル。ただし「このカードのゲーム上の役割」を理解する視点がなく、プレイヤーが自分で効果テキストを解釈する前提のツール。

---

### 2-C. ポケミミ（pokemimi.com）

**強み**（2026-03-25 実測確認）
- カード名キーワード検索
- カード種別（ポケモン / グッズ / サポート / エネルギー）フィルター
- HP範囲（最小〜最大）フィルター
- 12タイプすべてのチェックボックス選択
- **特性テキスト検索**・**ワザ名検索**（当初「弱い or 不明」と記載していたが実装済みと確認）
- 弱点・抵抗力ドロップダウン
- 進化段階・にげるコストフィルター
- グリッド / リスト 表示切替・人気順/ABC順ソート
- 横断検索（カード + ボックス情報）
- 40件/ページ・93ページ（約3,700件表示）

**弱み / ギャップ**
- **ワザ効果テキスト（フルテキスト）の検索は不明**（ワザ名検索はあるが効果文検索は不明）
- メカニクスグループ検索なし
- Q&A / ルール統合なし
- AND / OR / NOT 複合検索非対応

---

### 2-D. カードラッシュメディア（cardrush.media）

**強み**
- HP範囲・攻撃名・特性テキスト検索
- ワザ / 特性名で検索可能
- 進化段階・タイプ・弱点・抵抗力・にげるコストフィルター
- 「人気順」ソート（買取業者ならではの需要データ）

**弱み / ギャップ**
- メカニクスグループ検索なし
- Q&A統合なし
- テキスト AND / OR / NOT 検索非対応

---

### 2-E. DaiData 特性タグ検索（daidata.sakura.ne.jp）

**強み（最も我々のコンセプトに近い）**
- 13種のメカニクスタグで多選択フィルター可能：
  ドロー / サーチ / エネ加速 / 回復 / 打点UP / ダメ軽減 / 妨害 / 入れ替え / 回収 / LO / ばらまき / ベンチ撃 / 状態異常
- 特性名・効果テキスト検索（ひらがな ⇔ カタカナ変換対応）
- URLパラメータでフィルター状態共有可能
- カード件数表示

**弱み / ギャップ**
- **特性のみ対象**（ワザは対象外）
- ポケモンカードのみ（トレーナーズ・エネルギー非対応）
- Q&A / ルール統合なし
- ルール上の区別を反映したサブカテゴリ分類なし（例: ベンチ狙撃のダメージ型 vs ダメカン型）
- 旧シリーズのみ（現行レギュレーション未対応の可能性）

---

### 2-F. Limitless TCG（limitlesstcg.com/cards/jp）

**概要:** 英語主体のグローバルポケモンカードデータベース。日本語カードにも対応。業界最高水準の検索構文と大会メタデータを提供。

**強み**
- **最強クラスの検索構文（クエリ言語方式）:**
  - `text:キーワード` — ワザ名・特性名・効果テキスト全文検索
  - `hp:<=120`, `attack:>=3` — 比較演算子（<, >, <=, >=）対応
  - `has:ability`, `has:ancient-trait` — 特性有無フィルター
  - `weakness:fire`, `resistance:water`, `retreat:<=2` — 比較演算子付き弱点/抵抗力/逃げコスト
  - `regulation:`, `format:jp-standard` — レギュレーションマーク・フォーマット指定
  - `is:ace-spec`, `is:ex`, `is:full-art`, `is:alternate-art` — カード種別・アート種別
  - `banned:jp-standard` — 禁止カードフィルター
  - `unique:text` — 効果テキスト重複排除
  - `sort:`, `display:`, `show:` — UI表示制御
- **大会メタデータ（日本語スタンダード対応）:**
  - デッキシェア率（例: Dragapult ex 21.12%）
  - フォーマット・期間・大会種別・地域・部門別絞り込み
  - シティリーグ・地区大会・チャンピオンシップ結果DB
  - プレイヤープロファイル・ランキング
- **付帯ツール:** DrawCalc（確率計算）/ Swiss Calculator（トナメ管理）/ Proxy印刷

**弱み / ギャップ（我々との差）**
- **インターフェース英語主体**：日本語競技プレイヤーに馴染みにくい
- **メカニクスグループ検索なし**：強力な検索構文を持つが「グループ」概念はなくユーザーが構文を知る必要がある
- **Q&A / ルール統合なし**：カード詳細から公式Q&Aへの導線なし
- **日本語カード収録に遅延の可能性**：最新弾は日本版リリース直後に収録されない場合あり

**ポジション:** 英語圏向けのプロ仕様検索ツール。大会メタ分析は最強だが日本語プレイヤーには使用障壁が高い。

---

### 2-G. ポケカジラ（pokecazilla.com）

**概要:** 複数カードショップの買取・販売価格比較サイト。2024年以降カード検索機能を強化。2026年時点でポケモン・ワンピース・遊戯王・MTG・デュエマの5TCGに拡大。

**強み**
- カード名・HP値・特性名・ワザ名・効果テキスト全文検索（2024年アップデート後）
- レアリティ・レギュレーション・パック別フィルター
- 複数ショップ間の価格比較・価格トレンド追跡

**弱み / ギャップ**
- **目的が価格比較**：カード調査ツールとしての設計ではなく、テキスト検索はあくまで補助機能
- **メカニクスグループ検索なし**
- **Q&A / ルール統合なし**
- **AND / OR / NOT 検索非対応**

**ポジション:** 「カードを買う」ユーザー向け。カードを「理解する・調べる」用途は範囲外。

---

### 2-H. PokecaBook（pokecabook.com）

**概要:** ブログ/メディア系サイト。ルール解説記事・デッキ記事・大会結果・ティアリストを提供。

**強み**
- ルール裁定解説記事カテゴリ「ルール【裁定解説】」：特定カード（ポピーの裁定など）をテーマにした手書き記事
- デッキレシピ・メタ解説・初心者向けガイド
- Extra フォーマットの禁止カード情報記事

**弱み / ギャップ**
- **ルール情報が記事ベース**：構造化データではなく全文検索・カード名絞り込み不可
- **カードDB なし**：カード検索機能なし
- **Q&A DB なし**：公式Q&Aへの統合なし
- 記事数に依存するため網羅性に限界

**ポジション:** 「読む」メディア。調べる・比較するツールとしては機能しない。

---

### 2-I. ポケカデータラボ（pdatalab.com）

**概要:** Twitter/X から大会結果を自動収集してダッシュボードを提供するメタ分析専門プラットフォーム。

**強み**
- 大会結果（シティリーグ等）の自動収集・デッキシェア率・勝率統計（Looker Studio ダッシュボード）
- イベント検索（ポケモンカード対戦イベント検索 EX）
- 大会用スイスドロー・ペアリングツール

**弱み / ギャップ**
- **カード検索なし**：カードデータベース機能なし
- **Q&A / ルール情報なし**
- 完全にメタ競技分析ツールであり、カードを「理解する」用途とは無関係

**ポジション:** 競技プレイヤー向けのメタ分析専門ツール。我々のターゲット（ルール理解・カード調査）とは棲み分けが明確。

---

### 2-J. pcg-search（pcg-search.com/card/search.php）

**概要:** 旧シリーズ（WEB・ADV・PCG 等）専用カード検索サイト。現行レギュレーションは範囲外。

**強み**
- 旧シリーズのカード名・効果テキスト・イラストレーター・HP・タイプ・にげるコスト・ポケパワー/ポケボディ検索
- Twitter/X ログインによるデッキ保存・共有機能
- 現行ツールがカバーしない旧弾を網羅

**弱み / ギャップ**
- **現行レギュレーション（スタンダード/エクストラ）対応なし**
- メカニクスグループ検索なし・Q&A統合なし

**ポジション:** 旧シリーズ愛好家・ヴィンテージ研究向け。現行プレイヤーとのオーバーラップは小さい。

---

### 2-K. ポケカードラボ（pokecardlab.com）

**概要:** 2018年〜現在の大会デッキレシピを収録した競技向けアーカイブメディア。

**強み**
- シティリーグ・チャンピオンズリーグ・ジムバトル結果のデッキレシピ閲覧
- カード名からそのカードが入賞したデッキを逆引き可能
- 初心者向けガイド・戦略コラム・カードパズル

**弱み / ギャップ**
- **カードDB なし**：カード検索機能なし、メカニクス分析なし
- ルール/Q&A 情報なし
- 「カードを理解する」ではなく「デッキ構築の参考にする」用途

**ポジション:** デッキビルダー向けレシピアーカイブ。我々の用途（カード特性理解・ルール調査）とは棲み分け明確。

---

### 2-L. ポケカWINデッキス（pokeka-win-decks.jp）

**概要:** シティリーグ等の大会結果をX（Twitter）から自動収集し、日次更新のティア表として提供。

**強み**
- 最新の環境ティア表（デッキシェア率・勝率）を自動更新
- アーキタイプ別の採用率ランキング

**弱み / ギャップ**
- カード検索・テキスト検索・ルール情報は一切なし
- 純粋なメタデータ閲覧ツール

**ポジション:** 競技プレイヤーの環境把握に特化。カード調査・ルール理解とは無関係。

---

### 2-M. 初手わかる君 / ひとり回し君（Chrome 拡張）

**概要:** 公式サイト（pokemon-card.com）上でソロプレイテストを可能にする非公式 Chrome 拡張機能群。

**強み**
- デッキを物理的に組まなくても公式UI 上でシミュレーション可能
- pnote の1人回し機能と同等の用途を PC ブラウザで実現
- ドロー・入れ替え・コイントスを自動処理

**弱み / ギャップ**
- Chrome 拡張のみ（Safari / Firefox 非対応、モバイル不可）
- カード検索・メカニクス分析・ルール情報は持たない
- 公式サイトの UI 変更で動作不能になるリスクあり

**ポジション:** プレイテスト専用。「カードを理解する」ツールとは完全に別カテゴリ。

|---|---|---|---|---|---|---|---|---|---|

## 4. 我々のツールの差別化ポイント（優先度順）

### [最重要] D-01: ルール根拠のメカニクスグループ検索

**現状の競合:** 存在しない。DaiDataが特性のみ13タグで近いが、ワザ非対応、サブカテゴリなし。ポケカ検索くんはフィルター検索が豊富だが「グループ」概念がなくユーザーが自分でテキストを解釈する必要がある。pnoteはグループ検索機能を持たない。

**我々の実装計画:**
`group_categories_proposal.md` に定義した30+グループを検索UIに実装。
- ベンチ狙撃（ダメージ型 / ダメカン型 / 移動型）
- ポケモン入れ替え（自分 / 相手C-04型 / 相手C-05型）
- 手札干渉（トラッシュ / 山札戻し / 手札を見る）
- エネ加速（山札→手札 / 山札→ポケモン / 手札→ポケモン / トラッシュ→ポケモン）
- 特殊状態（どく / やけど / こんらん / ねむり / マヒ）
- その他（にげられない / 自傷ダメカン / ドロー / 山札破壊 / 特性無効化 等）

**価値:** 「ベンチ狙撃への対策カードを全部見たい」「入れ替え効果があるカードを種類別に把握したい」というプレイヤーニーズに答える唯一のツール。

---

### [重要] D-02: Q&A とカード詳細の双方向リンク

**現状の競合:** 公式FAQはカード名検索できるが、カードDBと分断。他サイトはQ&A自体なし。

**我々の実装計画:**
- カード詳細モーダルに「このカードのQ&A」セクションを追加（`qa_card_map.json` 活用）
- Q&A一覧から該当カードの詳細に飛べるリンク
- メカニクスグループ + Q&Aを組み合わせた「このメカニクスに関するよくある質問」ビュー

**価値:** 「このカードのベンチ狙撃に弱点/ダメカン軽減は適用されるのか?」という疑問をカード詳細から即座に解決できる。

---

### [重要] D-03: AND / OR / NOT テキスト検索

**現状の競合:** 全サイト非対応。

**我々の実装:** 既に実装済み（スペース区切りでAND、`-term`でNOT、OR切替トグル）。

**価値:** 「ダメカン かつ ベンチ を含むワザ」「こんらん OR ねむり を付与するカード」等の複合検索。

---

### [差別化] D-04: ワザ vs 特性の統合テキスト検索

**現状の競合:** DaiDataは特性のみ。カードラッシュは一部対応。

**我々の実装:** ワザ・特性・ルールテキストを統合した全文検索。既に実装済み。

---

### [差別化] D-05: ルール解説の文脈付きグループ説明

**現状の競合:** なし。

**我々の実装計画:**
各メカニクスグループページに「なぜこれが重要なのか」「よくある誤解」を `group_categories_proposal.md` のルール区別テーブルから自動生成。

例：ベンチ狙撃グループページに「ダメカン型はダメカン軽減・弱点/抵抗力が適用されない」旨を明記。

## 6. まとめ

**我々のツールが唯一提供できるもの:**

1. **ルール根拠のあるメカニクスサブカテゴリ検索**（C-04型 vs C-05型、ダメカン型 vs ダメージ型）— 他に存在しない
2. **Q&AとカードDBの双方向統合**（カード詳細 → 該当Q&A）— 他に存在しない
3. **全文 AND/OR/NOT テキスト検索**（ワザ + 特性 + ルール統合）— 他に存在しない
4. **トレーナーズを含む全カードタイプのメカニクスタグ化**— DaiDataは特性のみ

**各競合との棲み分け:**

| 競合 | 彼らが強い | 我々が補完する |
|---|---|---|
| pnote | デッキ管理・1人回し・大会記録（モバイル） | PC Web でのカード調査・ルール理解・グループ検索 |
| ポケカ検索くん | 詳細フィルター検索（HP/コスト/レギュ） | グループ概念・Q&A統合・ルール解説文脈 |
| 公式FAQ | 公式Q&A（信頼性） | カードDBとの統合・グループ横断ビュー |
| DaiData | 特性タグ検索 | ワザ対応・トレーナーズ対応・サブカテゴリ |
| Limitless TCG | 強力な構文検索・大会メタ分析（英語圏） | 日本語UI・グループ概念・Q&A統合・初心者フレンドリー |
| ポケカジラ | 価格比較・購入ナビ | カードを「理解する」ための調査・ルール文脈 |

競合との比較において、「デッキを理解したい・ルールを理解したい中〜上級プレイヤー向けのルール文脈付き検索ツール（PC Web）」というポジションが明確に空いており、我々のツールはそこを狙うべきである。pnote はデッキ操作ツールとして補完関係にあり、直接競合ではない。
