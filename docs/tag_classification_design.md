# Tag Classification Design

**Confirmed:** 2026-05-03

## Architecture decision

**Old approach**: Pre-tag cards in `card_tags.json` using text patterns → frontend filter by tag  
**Current approach**: `official_classifications.json` (config) → `build_official_class_index.mjs` (build) → `official_class_index.json` (index) → frontend filter

**Why**: Card text uses official category names directly (e.g. 「ポケモンex」), so tagging was redundant. Index-based approach is maintenance-free and handles new card additions automatically.

### Hardcoded lists

- **テラスタル**: 33 cards (all ex cards). Full list in `data/official_classifications.json`
- **古代**: 10 Pokémon + Koraidon variants (3) + 2 Trainers (覚醒のドラム, 探検家の先導)
- **未来**: 10 Pokémon + Miraidon variants (3) + 2 Trainers (リブートポッド, 暗号マニアの解読)

**Miraidon / Koraidon disambiguation (by move name)**:
- Koraidon「げんせいらんだ」「はじょうもうこう」→ 古代
- Koraidon ex「ほうふくてっつい」→ 古代
- Miraidon「アクセルピーク」「プロテクトコード」→ 未来
- Miraidon ex「リジェクトボルト」→ 未来
- Koraidon ex「ひひいろのキバ」→ テラスタル
- Miraidon ex「スラッシュクロー」→ テラスタル
- Koraidon「かくごのキバ」/ Koraidon ex「ツメできりさく」/ Miraidon「でんきのツメ」→ no tag

### Prefix search (card name matching)

Tags generated by prefix: ロケット団, Nのポケモン, ホップのポケモン, シロナのポケモン, ヒビキのポケモン, マリィのポケモン, アオキのポケモン, リーリエのポケモン, ナンジャモのポケモン, ペパーのポケモン, ダイゴのポケモン, エリカのポケモン, カスミのポケモン

Notes:
- 「ロケット団」tag name omits「のポケモン」suffix — includes both Pokémon and Trainers
- Prefix hits include non-Pokémon cards (bags, headbands, supporters) — intentional
